## Solution Setup

- [ ] Create asset directory: `mkdir -p assets/uscia-agent/`

- [ ] Validate `solution.yaml` and `assets/uscia-agent/asset.yaml` exist and are well-formed

## API Specs

> S/4HANA EDMX spec files are required for MCP translation. Pre-signed S3 download URLs expire after 3600 seconds — re-run `sap_knowledge_graph_api_discovery` immediately before downloading at execution time.

- [ ] Call `sap_knowledge_graph_api_discovery` for fresh EDMX download URLs (6 APIs: Planned Order, Material Planning, PIR, PP/DS Stock, PP/DS Constraints, Advanced ATP)
- [ ] Download 6 EDMX files to `specification/uscia-agent/api-specs/` immediately using Python `urllib.request`
- [ ] Invoke `mcp-translation-file` skill to generate MCP translation files for all 6 downloaded EDMX specs
  - **If `mcp-translation-file` or `generate_mcp_translation` is unavailable**: skip and log `[MCP-SKILL] mcp-translation-file unavailable — agent will call S/4HANA APIs directly via MISSING_DATA fallback pattern`
- [ ] Invoke `setup-solution` skill to register generated MCP server translation assets (skip if mcp-translation-file was skipped)
- [ ] Generate `mcp-mock.json` using `mcp-mock-config` skill (skip if no MCP specs exist)
