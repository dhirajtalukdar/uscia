Installation

Requires Python 3.10+ and uv. Works on macOS, Linux, and Windows.

MCP Server
  1. Clone the repository: 
    git clone https://github.tools.sap/I520242/ibp-bud6-mcp.git
cd ibp-bud6-mcp
  2. Install dependencies: 
    pip install -e .
  3. Copy and fill in your SAP credentials: 
    cp .env.example .env
# Edit .env with SAP_BASE_URL, SAP_USERNAME, SAP_PASSWORD
  4. Build the rules database: 
    python -m sap_ibp_abap_int.rules_ingest
  5. Register with Claude Code and start: 
    claude mcp add SAP-IBP-ABAP-INT -s user \
  -- sap-ibp-abap-int --env-file /path/to/.env
Configuration

Minimum .env for live SAP access:

SAP_BASE_URL=https://your-system.example.sap
SAP_USERNAME=your_user
SAP_PASSWORD=your_password
SAP_VERIFY_SSL=false

Add these for AI-powered tools:

LLM_PROXY_URL=http://localhost:6655/litellm/v1/chat/completions
LLM_API_KEY=your-bearer-token
LLM_MODEL=anthropic--claude-4.7-opus

Installation

Build the VSIX from source and install it into VS Code. Requires Node.js 18+ and npm.

Build & install the extension
  1. Clone the repository: 
    git clone https://github.tools.sap/I520242/ibp-abap-int-extension.git
cd ibp-abap-int-extension
  2. Install dependencies and build the VSIX: 
    npm install
npm run build:vsix
  3. In VS Code: Ctrl+Shift+P → Extensions: Install from VSIX…
  4. Select the generated .vsix file and reload VS Code when prompted
Prerequisites
  1. Install and start the MCP server first (see the MCP Server tab)
  2. The extension auto-connects on startup — ibpAbap.autoConnect is true by default
  3. For AI toolbar tools, add LLM_PROXY_URL, LLM_API_KEY, and LLM_MODEL to the server's .env
  4. For live syntax check, the SAP system must have a TEST_SYNTAX_CHECK package under $TMP

The rules database (rules.db) contains 2 795 tagged rules from IBP Conventions (13 topics), CleanABAP (134 sections), and IBP domain documentation (2 646 chunks). Each rule has a source, category (naming/methods/exceptions/etc.), and purpose (write/review/test). get_rules(query, purpose) uses FTS5 to return the most relevant rules before any code task. Team-specific rules can be added via the VS Code custom rules editor and are stored in rules.json.
Cached objects have a configurable TTL (default 24 hours). Classes use a lightweight version-list check — if the SAP version list changed, the object is re-fetched. Non-class objects (tables, CDS views, structures) are re-fetched when their TTL expires. diff_with_sap and get_sap_source opportunistically update the index when they detect differences — zero extra round-trips.
